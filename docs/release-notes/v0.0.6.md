# Open Island Feishu v0.0.6

## ⚠️ 重要 / Important — 首次安装必读

> 这个版本**没有 Apple Developer ID 公证**，只做了 ad-hoc 签名。如果你是从飞书 / 微信 / QQ / Telegram 等 IM 应用收到的 dmg/zip：
>
> 1. 双击 `Open Island Feishu.dmg` 挂载
> 2. 把 **`Open Island Feishu.app`** 拖进 **`Applications`**
> 3. **右键点击** `Fix-Launch.command` → 选 **「打开」** → 在弹框点 **「打开」** 确认
>    - 必须用右键打开；直接双击会被 macOS 拦下来
> 4. Terminal 会自动跑完修复脚本并启动应用，按任意键关闭窗口
>
> 直接双击 `.app` 会失败（崩溃 25ms 内、看不到任何报错）。原因是 macOS Sonoma+ 对从 sandbox 应用分发的未公证应用会直接拒绝执行，与本应用代码无关。
>
> ---
>
> This build is **not notarized with an Apple Developer ID** — it is ad-hoc signed only. If you received the dmg/zip via a sandboxed IM app (Feishu, WeChat, QQ, Telegram, ...):
>
> 1. Double-click `Open Island Feishu.dmg` to mount it
> 2. Drag **`Open Island Feishu.app`** into **`Applications`**
> 3. **Right-click** `Fix-Launch.command` → choose **Open** → confirm **Open** in the dialog
>    - You must use right-click; a plain double-click will be blocked by macOS
> 4. Terminal will run the repair script and launch the app; press any key to dismiss
>
> Double-clicking the `.app` directly will fail (the process is killed within 25 ms with no visible error). This is macOS Sonoma+ rejecting unnotarized apps delivered through a sandboxed source, not a bug in this app.

## 修复 / Fixes

### 修复"装上点开闪退"（从飞书等 sandbox 应用分发场景）

v0.0.5 因为 ad-hoc 签名 + `com.apple.quarantine`（由飞书 / 微信等 sandbox app 写入）+ macOS Sonoma+ Gatekeeper 收紧组合，导致用户**双击应用图标后被内核 kill 在启动 25ms 内**（崩溃报告里看到 `codeSigningTrustLevel: 4294967295`、`spctl: File created by an AppSandbox, exec/open not allowed`）。

修复方式：发布产物里多带一个 **`Fix-Launch.command`**：

- 剥掉 `com.apple.quarantine` 隔离属性
- 重新做 ad-hoc 签名
- `open` 启动应用

直到我们拿到 Apple Developer ID + 走完 notarytool 公证流程之前，这是必经步骤。

---

Fixes a launch failure on v0.0.5 when the app was distributed through sandboxed IM clients: ad-hoc signature + `com.apple.quarantine` (set by Feishu/WeChat/etc.) + tightened macOS Sonoma+ Gatekeeper caused the kernel to kill the process within 25 ms of launch (crash report shows `codeSigningTrustLevel: 4294967295`, `spctl: File created by an AppSandbox, exec/open not allowed`).

A `Fix-Launch.command` helper now ships beside the .app; double-click it (right-click → Open on first run) to strip quarantine, re-apply ad-hoc signing, and launch. This stays in place until we ship a Developer-ID-signed + notarized build.

## 已经装了 v0.0.5 卡住的用户

打开 `Terminal.app`，复制这一行回车，应用就能开了：

```bash
xattr -dr com.apple.quarantine "/Applications/Open Island Feishu.app" && open "/Applications/Open Island Feishu.app"
```

Already stuck on v0.0.5? Open Terminal and run:

```bash
xattr -dr com.apple.quarantine "/Applications/Open Island Feishu.app" && open "/Applications/Open Island Feishu.app"
```

## 从 v0.0.5 升级

**已经成功跑起来的 v0.0.5**：应用内 **设置 → 关于 → 检查更新**，Sparkle 自更新走 zip 不触发 quarantine 问题，无需手工修复。

**没能成功打开 v0.0.5 的用户**：按上面"重要"段落手工装 v0.0.6。
